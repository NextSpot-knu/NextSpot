## 0. 검증에서 확정된 전제 3가지

1. **심사 임팩트의 실제 변별점은 데모 대본 노출 여부다.** 머천트 콘솔·안전 관제·문의·실험실은
   `DEMO_SCENARIO.md`·`JUDGE_QA.md` 어디에도 등장하지 않는다(grep 0건). 데모에 실노출되는 AI 접점은
   관광객 /main 흐름과 관제 대시보드(0:20~0:50 KPI)뿐이다.
2. **JUDGE_QA 서사 정합**: 심사 방어 논리가 "실시간 프록시 호출이 아니라 사전 배치 적재+캐시"
   (`JUDGE_QA.md:61-62`)이므로, 신규 지점도 **요청 경로를 LLM에 블로킹시키지 않는다**
   (배치·장기 캐시·후행 호출·온디맨드 버튼 중 택일).
3. **콘솔 로케일 관례**: admin·merchant 콘솔은 한국어 하드코딩 전용(useT 미사용)이라 한국어 단일
   산출물이 관례에 맞다. 관광객 노출 문자열만 ko/en/ja/zh 4로케일 의무.

## 1. 공통 설계 원칙 (전 지점 적용)

LLM이 앉을 수 있는 자리는 3층뿐이다 — 양안 합의:

| 층 | 허용 | 예 |
|---|---|---|
| ① 구조화 | 자연어 → **기존 enum·화이트리스트** 코드 (신규 라벨 창작 금지) | 선호 파싱, 검색어 재작성 |
| ② 요약·설명 | **서버가 계산한 사실**의 프로즈화 (숫자·시설명은 서버가 원천) | 브리핑, 축제 요약 |
| ③ 배치 정제 | 원문 보존 + `*_ai_*` 별도 키 후보 생성, 자동 확정 금지 | 번역·요약 배치 |

- **무해 폴백**(불변): `UPSTAGE_API_KEY` 미설정 → 자동 비활성. 타임아웃/HTTP/파싱 실패 → None →
  기존 결정적 경로. LLM 장애가 기능 장애·오정보로 승격되는 경로가 하나라도 있으면 설계 반려.
- **정직성 검증 게이트**(출력 채택 전 필수): 출력의 모든 숫자가 입력 JSON 의 부분집합인지 검증
  (`reason_service._is_honest_polish` 패턴). 추세어("증가/감소/완화")는 서버가 계산한 비교 필드가
  입력에 있을 때만 허용. 실패 시 전량 폐기 → 폴백.
- **보안 관례 계승**(§-14 하드닝): 프롬프트 내 사용자 입력은 json.dumps 데이터 경계, 원문·응답
  본문 로그 금지(길이만), TTS 직발화 금지, 화이트리스트 재검증.

## 2. 확정 5종

### P0-1. 자연어 선호 파싱 LLM 백스톱 — `/preferences/parse`

**양안 1위 일치.** "조용하고 사람 적은 데서 쉬고 싶어" 같은 키워드 사전 밖 문장이 현재는 빈 결과.

- **결정적 발견**: 프런트 `explore/recommend/page.tsx:536` 이 이미 `result.isFallback` 으로
  `nlAppliedAi` vs `nlAppliedKeyword` 토스트를 분기하고 4로케일 키도 존재하는데, 백엔드
  `preference_nlp_service.py:135` 가 `is_fallback=True` 고정이라 **nlAppliedAi 는 도달 불가한 죽은
  코드**다. 이 기능은 "죽은 UI를 살리는" 작업이다.
- **동작**: 키워드 매칭 성공 → 기존 경로 그대로(주 경로 불변). 키워드 전량 미스일 때만 Solar 호출.
  출력은 허용 카테고리·속성 화이트리스트 코드만 → `_coerce`(`:92`) 재검증 → 기존
  `build_preference_vector()`(`:54`) 가 벡터 생성. **LLM 이 8차원 벡터를 직접 출력하는 설계 금지.**
- **조건(검증에서 확정)**: ① LLM 성공 시 `is_fallback=False` 반환 + docstring(`:8`,`:115`) 갱신
  ② 사용자 노출 요약문(`_build_summary`, `:65`)이 한국어 단일이므로 4로케일 대응 필수 — 코드 산출
  라벨을 프런트 t() 키로 매핑(권장) ③ 비-ko 문장은 키워드 사전이 한국어 전용이라 LLM 경로로 몰림을
  전제로 비용 추정 ④ 포지셔닝은 음성 백스톱(기존 ①)의 텍스트 대응물 — 데모 센터피스 아님.
- **폴백**: 실패 → 현재와 동일한 키워드 빈 결과 → 콜드스타트 평균 벡터. **비용**: 인증+키워드 미스
  시에만, 건당 소액. 지연 0.8~1.1s 는 기존 `nlAnalyzing` 로딩 UI가 흡수.
