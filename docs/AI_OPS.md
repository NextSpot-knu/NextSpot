# AI 운영 플레이북 — Claude Code · Codex CLI · Antigravity

멀티 AI CLI 운영의 정본. 지침 파일 규약, 도구별 역할, 게이트, 핸드오프 규칙을 정의한다.
프로젝트 자체의 규칙(검증 게이트·가드레일)은 루트 `AGENTS.md`가 정본이며, 이 문서는 "도구를
어떻게 조합해 쓰는가"만 다룬다.

## 1. 지침 파일 규약 — 정본 1개 원칙

| 파일 | 역할 | 읽는 도구 |
|---|---|---|
| `AGENTS.md` (루트) | **정본.** 컨벤션 변경은 항상 여기에만. | Codex(기본 규약), Claude·Gemini(임포트 경유), Antigravity(바이너리가 인식) |
| `apps/web/AGENTS.md` | 웹 하위 정본 (웹 전용 컨벤션) | 위와 동일 |
| `CLAUDE.md` | `@AGENTS.md` 임포트 포인터 | Claude Code |
| `GEMINI.md` | `@./AGENTS.md` 임포트 포인터 | Gemini CLI (import 처리기 내장 확인) |
| `.agents/rules/nextspot.md` | 워크스페이스 규칙 포인터 | Antigravity IDE (확장이 `**/.agents/rules/**/*.md`를 규칙 파일로 등록) |
| `docs/AI_OPS.md` | 이 문서 — 도구 조합·핸드오프 규칙 | 사람 + 필요 시 에이전트 |

포인터 3종(`CLAUDE.md`/`GEMINI.md`/`.agents/rules/nextspot.md`)에는 **내용을 직접 쓰지 않는다.**
도구가 자동 로드하는 경로가 서로 달라 포인터가 여러 개일 뿐, 실체는 `AGENTS.md` 하나다.
세 도구가 서로 다른 지침을 읽으면 컨벤션이 갈라진다 — 규칙이 생기면 `AGENTS.md`에만 쓴다.

## 2. 역할 분담

**기본 대형: "Claude가 짓고, Codex가 깐다."** Claude는 큰 그림(설계·통합·판단)에 머무르고,
검증과 문제 해결은 Codex로 내린다. 모델 계열이 달라 서로의 맹점을 잡는 것이 요점이다.

| 도구 | 형태 | 맡는 일 | 이유 |
|---|---|---|---|
| **Claude Code** | 헤드리스 CLI (`claude`) | **큰 그림**: 아키텍처·설계, 요구사항 분해, 구현 초안, 오케스트레이션, 게이트 최종 판단, `docs/HANDOVER.md` 갱신 | 대용량 컨텍스트로 저장소 전체 구조를 쥐고 감. 터미널·파일 시스템 실시간 인터랙션이 빠름 |
| **Codex CLI** | 헤드리스 CLI (`codex`) | **검증·문제 해결**: 교차 리뷰, 버그 추적·디버깅, 엣지 케이스, 테스트 작성, 기계적 수정 | 정밀 diff·추론에 강함. ChatGPT 구독 포함분이라 상시 게이트로 써도 부담 없음 |
| **Antigravity** | **IDE** (에이전트 매니저 + 내장 브라우저) | 브라우저 검증 전담: 데모 플로우 실클릭, 스크린샷·녹화 증거, 모바일 뷰포트 확인 | 내장 브라우저 에이전트가 차별점. Gemini 무료 쿼터로 검증 비용 절감 |

**디버깅은 Codex가 1차 대응**이다(PM 지시, 2026-07-15). Claude는 증상·범위·재현 조건을 정리해
넘기고, 돌아온 진단을 실환경에서 확인한 뒤 통합한다. Claude가 직접 가설·실험 루프를 오래 도는 것은
컨텍스트 낭비다 — 실제로 이 프로젝트의 `/waiting` 조사에서 Claude가 이미 해소된 401 유령을 좇아
토큰을 태운 전례가 있다.

**Antigravity는 헤드리스 CLI가 아니다.** 이 머신 기준 `Antigravity IDE/bin/antigravity-ide`는
워크스페이스를 여는 래퍼일 뿐이고, 실제 작업은 IDE의 Agent Manager에서 지시한다. 즉 §3의 게이트 ③은
파이프라인에 자동으로 끼우는 단계가 아니라 **사람이 IDE를 열고 돌리는 수동 검증**이다.

`gemini` CLI(0.50.0)는 설치·API 키 설정 완료. 키는 Git에서 제외된 `apps/api/.env`의
`GEMINI_API_KEY`에만 둔다. 선택적 읽기 전용 교차 검토는 루트에서 아래처럼 실행한다.

```powershell
.\scripts\gemini_review.ps1 "현재 변경분의 데이터 신뢰성·보안·회귀 위험을 검토"
```

래퍼는 프로세스 환경에 키가 없을 때 `.env`에서 값만 로드하고, 자동 실행을 위해 저장소 신뢰 확인을
건너뛰되 실제 도구 권한은 Gemini CLI의 `plan` 승인 모드로 고정한다. 결과는 반드시 실제 코드·테스트로
재검증하며 Gemini에게 직접 구현·커밋·푸시를 맡기지 않는다. 기본 모델은 지연·비용 대비 성능이 좋은
안정 버전 `gemini-2.5-flash`이며 필요할 때만 `-Model`로 명시적으로 바꾼다.

**핵심 원칙: 구현한 도구가 자기 코드를 리뷰하지 않는다.** 모델 계열(Anthropic/OpenAI/Google)이
달라 서로 다른 맹점을 잡는 것이 세 도구를 쓰는 가장 큰 이유다.

## 3. 표준 루프

1. **설계** — Claude Code. 계획은 대화에만 남기지 말고 `docs/HANDOVER.md` 또는 `docs/`에 내구 산출물로.
2. **구현** — `feature/jinseok`에서. 병렬 작업은 워크트리 격리 + 파일 소유권 분할
   (`docs/AUTONOMOUS_SESSION.md` 규칙). i18n(교차절단 문자열)은 병렬 금지 — 단일 소유.
3. **게이트 ① 결정적 검증** — `AGENTS.md` 검증 게이트 4종(web/pytest/ruff/스키마 파리티). 실패 시 커밋 금지.
4. **게이트 ② 교차 리뷰** — 구현 주체가 아닌 도구로: Claude가 구현했으면 `codex review`,
   Codex가 구현했으면 Claude `/code-review`. 발견은 적대적 재검증 후에만 수정(반증 전례: 127.0.0.1 CORS).
5. **게이트 ③ 브라우저 검증** (UI 변경 시, **IDE에서 수동**) — Antigravity IDE를 열어
   `docs/DEMO_SCENARIO.md`의 데모 전 체크리스트 플로우를 실클릭. 대상: `localhost:3000`
   (`.\run_local.ps1`) 또는 Vercel 프리뷰. 모바일 뷰포트(하단 safe-area·바텀내비) 포함.
   스크린샷을 증거로 남긴다. 공모전 데모 리허설이 이 게이트의 최종 형태다.
6. **머지** — `git push origin feature/jinseok:main` (fast-forward). main push가 곧 Vercel prod 배포다.

빠른 버그픽스는 ③을 생략할 수 있지만 ①·②는 생략하지 않는다.

## 4. 핸드오프 규칙

- **같은 워킹 트리를 두 도구가 동시에 편집하지 않는다.** 도구 간 인터페이스는 커밋/브랜치.
  동시 작업이 필요하면 git worktree로 분리.
- 하위 에이전트는 커밋만, **푸시는 통합자(메인 세션)가** 수행.
- 도구들은 서로의 대화 기록을 공유하지 않는다 — 세션 간 컨텍스트 전달 채널은
  `docs/HANDOVER.md`(상태 스냅샷 + 다음 단계 + 커밋 해시)가 유일하다. 어느 도구로 작업했든
  세션을 끝낼 때 HANDOVER를 갱신한다.
- 외부 계정이 필요한 작업(Supabase SQL Editor, Render/Vercel env, Kakao 콘솔, GitHub Secrets)은
  코드로 우회하지 말고 HANDOVER '사람 작업' 목록에 기록한다.

## 5. 자율(야간) 세션 — Claude Code 전용

무인 세션은 Claude Code로만 돌린다(다른 두 도구는 무인 장기 세션 운영 이력이 없음).

- 루프 규칙·RESUME 절차: `docs/AUTONOMOUS_SESSION.md`. 단 **무엇을 할지는 `docs/HANDOVER.md`가 정본** —
  AUTONOMOUS_SESSION의 "진행 상태"는 2026-07-10 로그라 그대로 이어받으면 완료된 일을 다시 잡는다.
- 결정적 게이트 우선 — 게이트 통과분만 커밋, 진행분 즉시 커밋·푸시로 유실 방지.
- 감사(audit)성 발견은 적대적 재검증을 통과한 것만 수정하고, 반려 항목도 HANDOVER에 기록.
- freeze 백로그(HANDOVER 명시) 항목은 자율 세션이 임의로 착수하지 않는다.

## 6. 쿼터 라우팅 — Claude 컨텍스트 아끼기

Claude 세션의 병목은 컨텍스트와 사용량 한도다. Codex(ChatGPT 구독)와 Gemini는 **별도 쿼터**를
쓰므로, 범위가 명확한 검토를 넘길수록 Claude의 한도가 남는다. Gemini는 모든 작업에 상시 호출하지
않고 대규모 변경, 데이터 신뢰성, 다국어·이미지 검토처럼 독립 관점의 가치가 큰 작업에만 사용한다.

### 핵심 원리

Claude가 `codex exec`를 부르면 **파일 읽기·탐색은 Codex의 컨텍스트에서** 일어나고,
Claude 컨텍스트에는 **돌아온 답만** 들어온다. 따라서 이득은 다음 경우에만 생긴다:

> **범위가 명확하고 근거를 file:line 으로 검증할 수 있는 "많이 읽고 조금 답하는" 조사 → Codex로.**
> 판단·설계가 필요한 일, 원문 근거 자체가 산출물인 일 → Claude.

"읽고 요약해 와"를 무조건 넘기면 안 된다 — 설계 입력, 보안 조사, 여러 파일에 걸친 불변식 확인처럼
**원문이 곧 근거인 작업은 짧은 요약이 핵심을 떨군다.** 돌아온 답을 file:line 으로 되짚을 수 있는
조사에만 적용한다.

실측(2026-07-15 이 문서를 만든 세션):

| 방식 | 한 일 | 비용 |
|---|---|---|
| Claude 하위 에이전트 4개 | 저장소 4개 영역 조사 | **209k 토큰 — 전부 Claude 쿼터** |
| `codex exec` 2회 | 변경분 교차 리뷰 | 76k 토큰 — **ChatGPT 쿼터**, Claude엔 답변 ~1.5k만 유입 |

앞의 저장소 조사(범위 명확·결과를 파일 경로로 되짚을 수 있음)야말로 Codex로 내렸어야 할 일이다.

### 넘길 것 / 안 넘길 것

**읽기 전용 오프로드 — 조건 없이 권장** (워킹 트리를 안 건드리므로 §4와 충돌 없음):

| → Codex | → Claude |
|---|---|
| 저장소 조사·탐색("X가 어디서 쓰이나") | 아키텍처 설계, 트레이드오프 판단 |
| 변경분 리뷰·감사 | 증상 정리·범위 확정(무엇을 넘길지 프레이밍) |
| **버그 추적·디버깅(가설→실험 루프)** | 돌아온 진단의 실환경 확인·통합 |
| 대량 로그·출력 분석 | 여러 발견을 종합해 결정 |

**쓰기 오프로드 — 조건부.** Codex에 파일 수정을 시키면 §4의 "같은 워킹 트리를 두 도구가 동시에
편집하지 않는다"에 걸린다. **직렬 실행**(Claude가 손 떼고 대기) 또는 **별도 worktree**로 격리하고,
끝난 뒤 게이트 + 교차 리뷰를 태운다. 규모는 §2 기준(작고 기계적인 수정)을 따른다.

### 호출 규약

`codex exec`의 스트리밍 출력은 통째로 Claude 컨텍스트에 들어오고 결과가 중복 출력된다.
**최종 답만 파일로 받아 필요한 부분만 읽는다.** Claude의 Bash 툴(POSIX sh) 기준:

```bash
cd /c/Users/hennr/Desktop/nextspot/NextSpot     # repo 안에서 실행 (아래 참조)
OUT="<scratchpad>/codex-$(date +%s).md"         # 매 실행 새 파일 — 재사용 금지
codex exec -o "$OUT" "<지시>" >/dev/null 2>&1; rc=$?
[ $rc -eq 0 ] || echo "codex failed rc=$rc"     # 실패를 삼키지 말 것
# 성공했을 때만 $OUT 을 Read
```

- `-o, --output-last-message <FILE>` — 최종 메시지만 파일로. **이게 기본 사용법이다.**
  (실측: 같은 리뷰가 스트리밍이면 수 KB 유입 → `-o` 면 2.5KB만)
- **고정 파일명 재사용 금지.** 실행이 실패하면 직전 실행 결과를 새 결과로 오인한다 —
  매 실행 새 파일 + 종료코드 확인이 한 쌍이다.
- **`--skip-git-repo-check` 는 불필요하다.** `NextSpot/` 은 정상 git repo다(repo 가 아닌 건
  중첩 폴더의 *바깥* `nextspot/`). repo 안에서 실행하거나 `-C <repo경로>` 로 루트를 지정할 것 —
  안전검사를 끄는 것보다 낫다.
- Codex는 **자체 샌드박스**에서 돈다 — 이 세션에서 `py -3.11` 이 Codex 환경에선 exit 103,
  실환경에선 정상이었다. **환경 의존적 발견은 실환경에서 재현 확인 후에만 수용.**

### 가장 싼 오프로드: Claude를 거치지 않기

Claude가 중계하면 왕복분(프롬프트+답변)은 여전히 Claude 쿼터다. 독립적인 큰 작업은
**별도 터미널에서 사람이 직접** Codex에 시키거나 Antigravity IDE에서 돌리는 편이 Claude 비용 0이다.
Claude는 설계·통합·최종 판단에만 부르는 것이 한도가 빠듯한 주간의 기본 전략이다.

어떤 라우팅이든 §3의 게이트는 동일하게 적용된다.
