# Solar 자율권 확장 기획 (2026-07-18 PM 승인)

Upstage Solar에게 어떤 자율권을 줄지의 기획 정본. PM과 합의된 원칙·우선순위·진행 상태를 기록한다.
세션 인계는 `HANDOVER.md`가 정본이고, 이 문서는 기획 상세다.

## 대원칙 — "Solar는 심사위원이 아니라 통역사다"

**판단(의미 해석)은 Solar, 실행·사실·순위는 서버.** 구체적으로:

- Solar 자율권은 **"무엇이 후보로 적격인가"**(자연어 의미 해석)와 **"어떻게 설명하는가"**까지.
- **"어떤 순서로 추천하는가"(랭킹)는 SPOT 산식의 독점 영역** — 절대 LLM에 넘기지 않는다.
  1. 공모전 방어력: SPOT은 breakdown(W1 0.4/W2 0.4/W3 0.2 수치 근거)으로 "왜 이 순서"에 답한다.
     LLM 랭킹은 설명 불가 + 출품작의 알고리즘 기여 소멸.
  2. 목적함수: 이 서비스는 맛집 추천이 아니라 **수요 재배치**(혼잡 분산)다. 재배치 인센티브는
     산식으로만 강제된다 — LLM은 "맛있어 보이는 곳"을 고르지 "분산시키는 곳"을 고르지 않는다.
  3. 재현성·정직성: 같은 입력=같은 순위여야 지표(수락률·분산 효과) 측정이 가능하고, 환각이
     순위에 개입할 통로가 없어야 한다.
- 파이프라인: **Solar(이해·적격 판정) → SPOT(순위) → Solar(설명)**.
- 안전 장치 관례(기존 확립, 신규 자율권에도 동일 적용): action/category 화이트리스트 enum,
  TTS 멘트는 서버 템플릿 전용(LLM spoken 폐기 — P1-1), 숫자 부분집합 정직성 검증
  (`_is_honest_polish`), 무인증 LLM 레이트리밋, 실패 시 결정적 폴백.
