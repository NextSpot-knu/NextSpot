# NextSpot 상용화 제품 아이디어 정본

> 2026-07-16 PM과 Codex의 제품 논의를 내구 문서로 정리한 기록이다. Claude Code, Codex,
> Gemini 및 다른 에이전트는 상용화 기능을 기획·구현하기 전에 이 문서를 먼저 확인한다.
> 이 문서는 아이디어/제품 방향 정본이며, 항목이 존재한다는 이유만으로 구현이 승인된 것은 아니다.

## 1. 상용화 판단

NextSpot은 혼잡 예측, 개인화 추천, 코스, 쿠폰, 음성 비서, 다국어, PWA, 상인 콘솔,
B2G 관제까지 이미 넓은 기능 표면을 갖고 있다. 상용화의 우선 과제는 기능 수를 늘리는 것이 아니라
다음 고객 루프를 신뢰할 수 있게 완결하는 것이다.

`추천 신뢰 → 실제 이동 → 도착 확인 → 쿠폰 사용/방문 평가 → 더 정확한 다음 추천`

### 출시 필수 후보(P0)

- 장소 정보 신뢰도: 실제 영업/휴무/브레이크타임, 가격대, 편의·접근성, 갱신 시각, 오류 신고,
  사업주 확인 배지.
- 추천 설명: 도착 예정 시각, 예상 혼잡/대기, 실제 보행시간, 취향 일치 근거, 예측 신뢰도,
  실측과 AI 예측의 명확한 구분.
- 방문 폐루프: 추천 수락, 길안내, 위치 또는 QR 도착 확인, 쿠폰 현장 검증, 방문 평가.
- 정식 계정: 카카오/네이버/Apple 등 간편 로그인, 게스트 기록 이전, 기기 간 동기화,
  탈퇴 및 개인정보 삭제.
- 현장 장애 대응: API/지도 장애 폴백, 위치 권한 거부 시 출발지 입력, 저장 코스·쿠폰 오프라인 열람,
  중복 쿠폰 방지, 임시휴업/통제 상태.

### 전환·재방문 후보(P1~P2)

- 상황형 추천: 동행, 남은 시간, 도보 한도, 예산, 실내/야외, 날씨, 유아·고령자 동반.
- 코스 재계산: 지연·혼잡·영업 종료에 따른 장소 교체, 특정 장소 고정 후 재추천, 출발/종료점 설정.
- 예약·웨이팅 연결: 전화/외부 예약 URL부터 시작하고 제휴점은 간편 대기 등록으로 확장.
- 일행 모드: 공유, 투표, 공동 편집, 구성원별 이동 제약 반영.
- 여행 전·중·후 연결: 일정 준비, 현장 재계산, 방문 지도·절약 시간·로컬 발견 여행 리포트.
- 통합 알림 센터: 한산 시점, 쿠폰 만료, 출발 시간, 휴업, 날씨 기반 코스 변경.

### 공급자 상용화 후보

- 상인: 사업자/매장 소유권 인증, 직원 권한, 실시간 영업·좌석·품절 상태, 쿠폰 QR 검증과 정산,
  유입→방문→사용 전환 퍼널, 청구·요금제·세금계산서.
- 지자체: 기관별 계정/RBAC, 감사 로그, 구역·기간·행사별 리포트, 개입 전후 비교,
  내보내기, 데이터 신선도/SLA, 다지역 관리.
- 현재 데모 토큰 기반 merchant/admin 인증은 정식 판매 전에 사용자별 인증으로 전환해야 한다.
- `merchant_timesales`와 좌석 상태는 `app/services/merchant_boost.py` 오버레이를 통해 추천 랭킹에
  반영된다(추천·코스·쿠폰 발급 3개 경로). 단 타임세일은 활성 세일 중 최댓값만, 좌석 상태는 30분 이내
  신선한 값만 적용된다. 연결은 됐으나 **효과 측정(발행→노출→방문 전환)은 아직 없다** — 효과를
  광고하기 전에 측정이 필요하다.

### 초기에는 미룰 후보

- 자체 리뷰 커뮤니티, SNS 피드, 복잡한 포인트 경제와 등급제.
- 위 기능은 어뷰징·운영 비용이 크고 핵심 가치인 혼잡 회피와 실제 이동을 직접 강화하지 않는다.

## 2. 제품 결정: 마이탭 `나의 실험실`

### 배경

사용자는 추천을 여러 번 연속 거절할 수 있다. 거절할 때마다 이유를 묻는 모달은 다음 추천을 빨리
보고 싶은 흐름을 방해하고 피로와 짜증을 만든다. 따라서 **거절은 즉시 처리하고 이유 입력은 선택적으로
마이탭에서 나중에 수행**하는 방향을 채택한다.

### 진입 카드

마이탭에 다음 성격의 카드를 둔다.

> 나의 실험실  
> 추천을 내 취향에 맞게 조율해요.  
> 피드백을 알려주면 NextSpot이 더 똑똑해지고, 다음 여행에서 시간을 더 아낄 수 있어요.  
> `추천 기록 다듬기 3건 →`

문구 대안: `알려줄수록, 더 잘 맞고 덜 기다리는 추천`.

### 거절 순간의 UX

- 거절 버튼을 누르면 이유를 묻지 않고 즉시 다음 추천을 보여준다.
- 최초 1~2회에 한해 `이유는 나의 실험실에서 알려줄 수 있어요`라는 비차단 안내를 짧게 노출한다.
- 기본 거절 신호는 저장하되, 상세 이유가 없다는 이유로 사용 흐름을 막지 않는다.

### 실험실 목록 UX

- 최근 거절 장소를 카드로 표시하고 추천 시각과 당시 핵심 조건을 함께 보여준다.
- 이유 선택 칩: `너무 멀었어요`, `너무 붐빌 것 같았어요`, `내 취향이 아니에요`,
  `가격이 부담돼요`, `영업하지 않았어요`, `이미 가본 곳이에요`, `지금 상황에 안 맞았어요`,
  `추천이 부정확했어요`, `기타`.
- 기타를 선택했을 때만 짧은 메모 입력을 연다.
- 전부 답하게 만들지 않는다. 개별 건너뛰기/목록 제거를 제공한다.
- 미응답 목록은 최근 5~10건으로 제한하고 30일 후 자동 만료한다. 장기간 쌓인 업무처럼 보이면 안 된다.
- 답변한 항목은 즉시 미처리 목록에서 제거한다.

### 학습 의미

모든 거절을 동일한 선호도 감점으로 처리하지 않는다.

| 이유 | 반영 방식 |
|---|---|
| 너무 멀어요 | 최대 이동시간/거리 민감도 조정 |
| 너무 붐빌 것 같아요 | 혼잡·대기 민감도 조정 |
| 내 취향이 아니에요 | 카테고리/특성 선호 조정 |
| 가격이 부담돼요 | 가격대 선호 조정 |
| 영업하지 않았어요 | 취향 학습 금지, 장소 정보 오류로 운영 큐 전달 |
| 이미 가본 곳이에요 | 취향 감점 없이 재추천 빈도만 제한 |
| 지금 상황에 안 맞았어요 | 장기 프로필보다 현재 세션/상황 신호로 반영 |
| 추천이 부정확했어요 | 모델/데이터 품질 검토 대상으로 분류 |

현재의 일률적인 거절 감점이 있다면 이유 입력 전에는 약한 임시 신호로만 다루고, 이유가 입력되면
의미에 맞게 보정하는 방향을 검토한다.

### 실험실 확장 방향

- 추천 기준 조절: 최대 도보 거리, 허용 대기시간, 가격대, 실내/야외, 숨은 장소 선호,
  익숙한 취향과 새로운 발견 비율.
- 학습 결과 설명: `도보 15분 이내를 선호해요`처럼 NextSpot이 이해한 내용을 보여주고 사용자가
  맞음/아님/삭제로 교정할 수 있게 한다.
- 반복 패턴 제안: `최근 거절 4건 중 3건이 도보 20분 이상이었어요. 앞으로 15분 이내를 우선할까요?`
- 효과 표시는 실측 근거가 있을 때만 제공한다. 근거 없이 `정확도 20% 상승` 같은 수치를 만들지 않는다.

## 3. 구현 시 필요한 데이터

거절 시점에 다음 스냅샷을 남겨야 나중에 이유를 정확히 해석할 수 있다.

- recommendation/user/facility 식별자와 추천 순위
- 추천·거절 시각, 당시 위치 또는 비식별화된 출발 구역
- 당시 예측 혼잡, 대기시간, 이동시간, 거리, 가격대(존재 시)
- SPOT 점수 및 score breakdown
- `rejected`, `reason_pending`, 상세 이유, 답변 시각
- 피드백 반영 범위(session/long-term/data-quality)와 처리 상태

개인 위치는 개인정보 최소수집 원칙을 적용하고, 실험실 기능·학습 활용·삭제 방법을 개인정보 화면에서
설명해야 한다.

## 4. 구현 전 확인 사항

- 기존 `user_feedback`과 `recommendations` 스키마 및 RLS로 위 데이터를 안전하게 저장 가능한지 확인.
- 현재 거절 시 적용되는 선호 벡터 `-5%` 로직과 상세 이유 후 재보정 시 중복 학습 여부 확인.
- 마이탭 카드는 미응답 건수를 서버 기준으로 표시하고, 실패 시 가짜 숫자를 만들지 않는다.
- i18n은 ko/en/ja/zh 4개 로케일을 동시에 반영한다.
- UI 변경 후 모바일 하단 내비게이션, 빈 상태, 30일 만료, 연속 거절 흐름을 브라우저에서 검증한다.
