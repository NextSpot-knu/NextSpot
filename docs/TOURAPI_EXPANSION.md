| 4 | **이야기 있는 대안 추천** | 포토코리아 관광사진 + Odii 음성·대본·사진 | 유명지를 포기해도 될 만큼 대안지의 매력과 방문 이유 제공 | 저작권·출처 표기, 위치/contentId 매칭, 경주 커버리지 실측 |
| 5 | **조건 충족형 안심 코스** | KorWithService + 반려동물 동반여행 서비스 | 휠체어·유모차·반려동물 조건을 만족하는 장소만 연결 | Boolean 하나로 축약 금지, 경주 표본 수가 적으면 기능 숨김 |
| 6 | **B2G 관광 다양성 KPI** | 지역별 관광객·소비·국제적 다양성 | 방문량뿐 아니라 연령·국적·소비 구성 다양화 평가 | 지역 집계값을 개인 취향 추정이나 프로파일링에 사용 금지 |

## 4-1. 여행 날짜 기반 30일 혼잡 회피 플래너

현재 NextSpot의 당일·도착시점 예측을 여행 전 단계로 확장한다.

- 사용자가 여행 날짜, 동행, 체류시간, 출발/종료점을 입력한다.
- 관광지 집중률 30일 상대 예보로 날짜별 위험도를 먼저 보여준다.
- 방문일이 고정됐으면 덜 붐비는 시간과 방문 순서를 제안한다.
- 인기 장소를 무조건 제거하지 않고 `인기 장소 오전 방문 → 오후 대안 코스`처럼 핵심 목적을 보존한다.
- 여행 전날 예측 변화나 행사 정보를 반영해 코스 재계산을 제안한다.
- 표시 문구는 `장기 집중 예상`과 자체 `도착시점 AI 예측`을 분리한다.

예시:

> 8월 3일 황리단길은 집중도가 높을 것으로 예상돼요. 오전 10시 이전에 방문하고,
> 오후에는 교촌마을→박물관 코스로 이동하면 예상 대기를 줄일 수 있어요.

## 4-2. 인기 동선에서 한 칸 옆 분산 그래프

관광지를 노드, 실제 연계 방문 관계를 간선으로 만들고 자체 혼잡 예측을 결합한다.

1. 중심 관광지 API로 수요가 모이는 허브를 찾는다.
2. 연관 관광지 API로 함께 방문되는 후보를 연결한다.
3. 동일한 여행 맥락을 유지하되 혼잡한 노드에 페널티를 준다.
4. 최종 이동시간은 보행/차량 경로 API로 다시 계산한다.
5. 결과에는 우회 비용과 절감 예상시간을 함께 설명한다.

예시:

> 첨성대 방문객이 함께 찾는 장소 중 지금 가장 여유로운 곳이에요. 일반 동선보다 12분 더
> 이동하지만 예상 대기시간은 24분 줄어들어요.

중심 관광지 데이터는 차량 목적지 조회 기반이므로 `사람들이 걸어서 이동한 동선`으로 설명하지 않는다.

## 4-3. 이야기 있는 대안 추천

혼잡하지 않다는 사실만으로는 유명 장소를 포기시키기 어렵다. `detailImage2`를 넘어 포토코리아와
Odii를 결합해 대안 장소의 방문 동기를 만든다.

- 추천 카드에 품질이 검증된 공식 관광사진과 촬영 키워드를 활용한다.
- 출발 전 30초 미리 듣기, 이동 중 오디오, 도착 시 위치 기반 이야기를 제공한다.
- 한국어·영어·중국어·일본어 공식 해설이 있으면 로케일에 맞춰 노출한다.
- 여행 후에는 방문 장소의 사진·이야기를 묶어 여행 기록으로 제공한다.
- 기존 음성 비서를 `명령을 듣는 도구`에서 `이동 중 여행 동행자`로 확장한다.

콘텐츠 매칭은 `contentId` 직접 일치가 없을 수 있으므로 좌표·정규화 장소명·거리 임계값을 조합하고,
확신도가 낮은 자동 매칭은 운영자 승인 전 노출하지 않는다.

## 4-4. 공식 다국어 장소 정보

- `name_{locale}`, `overview_{locale}`, 이용 안내, 공식 링크, 행사명과 이미지 메타데이터를 적재한다.
- 표시 우선순위는 `공식 해당 언어 → 공식 한국어 원문 → 명시된 AI 번역`으로 한다.
- 언어별 데이터 출처와 갱신 시각을 저장한다.
- 번역 데이터가 없을 때 해당 장소를 추천 후보에서 제거하지는 않는다.
- 외국인 로케일에서 한국어 축제명·운영안내가 그대로 삽입되는 현행 문제를 해소한다.

## 4-5. 조건 충족형 안심 코스

배리어프리·반려동물 정보를 단일 `가능/불가능` 값으로 축약하지 않는다.

- 휠체어 출입, 장애인 화장실, 엘리베이터, 유모차, 주차 등 세부 조건을 유지한다.
- 반려동물은 실내/실외, 이동가방, 크기 제한, 동반 구역, 유의사항을 구분한다.
- 사용자가 선택한 필수 조건은 랭킹 가점이 아니라 후보 제외 hard constraint로 처리한다.
- 데이터가 `미상`인 장소를 `불가능`으로 단정하지 않고, 확인 필요 상태로 표시한다.
- 경주 데이터 커버리지가 제품 임계값보다 낮으면 필터 자체를 숨기거나 베타 라벨을 붙인다.

## 4-6. `나의 실험실` 피드백과 결합

상용화 기획 정본 `docs/COMMERCIAL_PRODUCT_IDEAS.md`의 거절 이유를 TourAPI 데이터 품질과 연결한다.

| 거절 이유 | TourAPI 결합 처리 |
|---|---|
| 영업하지 않았어요 | 취향 감점 금지, 운영시간/동기화 정보 대조, 관리자 검증 큐 생성 |
| 추천이 부정확했어요 | 공식 분류·메뉴·접근성·동반 조건과 당시 추천 조건 재검사 |
| 내 취향이 아니에요 | TourAPI 세분류/공식 키워드 단위로 선호를 보정 |
| 이미 가본 곳이에요 | 취향 감점 없이 연관 관광지 그래프의 새로운 장소로 확장 |
| 지금 상황에 안 맞았어요 | 휴무·행사시간·이용시간·동반 조건을 확인해 세션 추천만 재계산 |

정보 오류 피드백은 개인화 신호와 데이터 품질 신호를 분리 저장한다. 동일 시설에 오류 신고가 반복되면
TourAPI 최신값 재조회 → 사업주 확인 → `운영정보 확인 필요` 표시로 이어지는 폐루프를 만든다.

## 4-7. B2G 관광 다양성 활용

지역별 관광 다양성은 개인 추천 피처가 아니라 정책 평가용 집계로만 사용한다.

- 특정 연령대 또는 국적에 편중된 권역 파악.
- 외국인 방문 대비 다국어 콘텐츠가 부족한 구역 식별.
- 분산 코스 노출 전후의 장소·권역 다양성 변화 측정.
- 추천 수락률/실방문과 함께 `관광객 구성 다양화`를 보조 KPI로 제공.
- 지역 집계값으로 개인의 연령·국적·소비 성향을 추론하지 않는다.

## 권장 실행 순서

1. 집중률 승인·경주 매칭률 확인 후 여행 날짜 플래너 얇은 버전 구현.
2. 연관/중심 관광지의 경주 표본과 시설 매칭률 측정 후 그래프 후보 생성기를 오프라인 배치로 구축.
3. 공식 다국어 데이터의 contentId 정합률을 20~30개 표본으로 확인.
4. 포토코리아/Odii의 경주 좌표·장소명 매칭률과 저작권 메타데이터를 확인.
5. 무장애/반려동물은 세부필드 커버리지가 임계값을 넘을 때만 고객 필터로 노출.
6. 각 기능은 추천 수락률뿐 아니라 길안내 진입·도착 확인·오류 신고까지 측정해 실제 분산 효과를 검증.
